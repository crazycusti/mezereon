# Tseng Labs ET4000 Graphics Controller Reference

## Documentation Reference
- **Original Datasheet (1990):** [http://bitsavers.informatik.uni-stuttgart.de/components/tsengLabs/Tseng_Labs_ET4000_Graphics_Controller_1990.pdf](http://bitsavers.informatik.uni-stuttgart.de/components/tsengLabs/Tseng_Labs_ET4000_Graphics_Controller_1990.pdf)

## Hardware Overview
The ET4000 is a high-performance Super VGA controller featuring a powerful internal architecture (64-bit interleaved memory bus in AX variant) and extensive backward compatibility with VGA/EGA/CGA/MDA.

## Key Register Map

### Unlock/Extension Controls
- **0x3BF (HERC/Extension Control):** Bit 0 enables/disables the Tseng extension register access via 0x3CD. Set to 1 to unlock.
- **0x3CD (Segment/Signature Register):** Used for bank switching (Bits 0-3 for read bank, 4-7 for write bank) in standard mode. Also used as a signature latch for detection (0x55/0xAA pattern).

### Banking (Segmentation)
The ET4000 uses a 64KB window at `0xA0000`. To access larger framebuffers (e.g., 640x480x8 @ 300KB), banking is required.
- **Register 0x3CB (Segment Select):** Alternative banking register.
- **Register 0x3CD (Segment Select):** Bits 0-3 select the 64KB read bank, bits 4-7 select the 64KB write bank.

### 256-Color (8bpp) Configuration
To enable 8bpp "chained" mode (Mode 13h / SuperVGA):
- **SR4 (Sequencer Register 4):** Bit 3 (Chain-4) must be set for linear 256-color addressing.
- **GR6 (Graphics Controller 6):** Bits 2-3 define memory mapping (typically `01` for `0xA0000` window).
- **AR10 (Attribute Controller 16):** Bit 0 (Graphics Mode) must be set. Bit 6 (Pixel Double) may be used for specific modes.

## Implementation Notes in Mezereon
- **Mode 12h (640x480x4):** Standard VGA planar mode (4 planes).
- **Mode 2Eh (640x480x8):** Custom Tseng SuperVGA mode. Requires manual banking via `0x3CB` during shadow buffer upload.
- **Shadow Buffer:** Mezereon uses a linear shadow buffer in main RAM and performs a banked upload to VRAM during `fb_sync`.

## Detection Logic
Mezereon identifies the ET4000 by:
1. Unlocking 0x3BF.
2. Toggling bits in the 0x3CD signature port (0x55/0xAA).
3. Fallback: Toggling CRTC register 0x33 (Tseng specific latch).

## VRAM Detection
SR0F (Sequencer Register 0x0F) contains memory configuration:
- Bits 3-4 define the amount of VRAM (256KB, 512KB, 1MB, or more).
- Mezereon uses this to decide whether to enable 8bpp modes by default (requires >= 512KB for 640x480x8).
